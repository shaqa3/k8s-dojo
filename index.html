RBAC · authorization simulator

Can I do that?

Kubernetes RBAC is allow-only and additive: a subject can act only if some Role or ClusterRole, bound to it, grants the verb on the resource. There are no deny rules. Build a question below and watch the answer — and exactly which binding decides it. Mirrors kubectl auth can-i, which you can also run in the Cluster Lab terminal.
